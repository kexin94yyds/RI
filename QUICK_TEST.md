## 🎯 测试步骤

### 步骤 1: 准备目标应用
打开以下任一应用作为拖拽目标：
- ✅ **文本编辑器**（推荐）: 打开 TextEdit 或 VSCode
- ✅ **微信**: 打开一个聊天窗口
- ✅ **备忘录**: macOS 自带的备忘录应用
- ✅ **Notion/Obsidian**: 如果已安装

### 步骤 2: 测试拖拽

1. **在应用中定位到历史记录列表**
   - 左侧是模式列表
   - 中间是历史记录列表
   - 右侧是预览区

2. **拖拽普通文本**
   - 用鼠标按住"这是一个普通文本测试"
   - 拖到文本编辑器
   - 松开鼠标
   - ✅ **预期**: 原样粘贴

3. **拖拽 URL 链接**
   - 拖动链接项到文本编辑器
   - ✅ **预期**: 显示为 `[https://github.com/electron/electron](https://github.com/electron/electron)`

4. **拖拽多行文本**
   - 拖动代码片段到文本编辑器
   - ✅ **预期**: 显示为代码块格式
   ````
   ```
   function hello() {
     console.log("Hello World!");
     return true;
   }
   ```
   ````

5. **观察视觉反馈**
   - 拖拽时，列表项应该变半透明
   - 显示蓝色边框
   - 状态栏显示"正在拖拽 Markdown 内容..."

### 步骤 3: 验证结果

在目标应用中检查：
- ✅ 内容是否正确粘贴
- ✅ Markdown 格式是否正确
- ✅ 是否保留了原始结构

## 🐛 如果遇到问题

### 问题 1: 拖拽没有反应
**检查**:
- 确认列表项上方有鼠标指针图标
- 尝试在不同应用中测试
- 查看开发者工具控制台是否有错误

### 问题 2: Markdown 格式不正确
**检查**:
- 打开开发者工具（已默认打开）
- 在 Console 中输入: `convertToMarkdown({ type: 'text', content: 'test' })`
- 查看转换结果

### 问题 3: 拖拽样式没有显示
**检查**:
- 刷新应用 (Cmd+R)
- 确认 style.css 已更新
